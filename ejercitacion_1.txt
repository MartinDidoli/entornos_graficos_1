1. ¿Qué es HTML, cuándo fue creado, cuáles fueron las distintas versiones y cuál es la última?
2. ¿Cuáles son los principios básicos que el W3C recomienda seguir para la creación de documentos con HTML?
3. En las especificaciones de HTML, ¿cuándo un elemento o atributo se considera desaprobado? ¿Y obsoleto?
4. ¿Qué es el DTD y cuáles son los posibles DTDs contemplados en la especificación de HTML 4.01?
5. ¿Qué son los metadatos y cómo se especifican en HTML?

1. HTML es un lenguaje de marcado para la elaboración de páginas web, define la estructura básica y un código para la disposición de contenidos. Comenzó a ser desarrollado en 1990 y la primera aparición pública fue a finales de 1991. Las distintas versiones fueron por orden de desarrollo: HTML, HTML 2.0, HTML 3.2, HTML 4.01, XHTML 1.0, HTML5, HTML 5.1, y la última versión lanzada fue HTML 5.2.

2. Los principios básicos que el W3C recomienda seguir para la creación de documentos con HTML son compatibilidad, utilidad, interoperabilidad, y acceso universal (https://www.w3.org/TR/html-design-principles).

3. En las especificaciones de HTML, un elemento o atributo se considera desaprobado cuando ha sido reemplazado por nuevos elementos. Y un elemento o atributo se vuelve obsoleto cuando no se garantiza que vaya a ser soportado en siguientes versiones (https://www.w3.org/TR/html401/conform.html).

4. El DTD es la declaración del tipo de documento y se ubica en la primera línea del archivo HTML. Los posibles DTDs contemplados en la especificación de HTML 4.01 son la declaración transitoria y la declaración estricta.

5. Los metadatos son declaraciones que describen datos, y HTML utiliza el elemento <meta> para especificarlos.
